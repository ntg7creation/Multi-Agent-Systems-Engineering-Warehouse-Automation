Environment  Characterization  
The  proposed  environment  is  a  grid-based  warehouse  map  of  size  X×YX  \times  YX×Y.  Each  
cell

●  a  free  road  cell  that  agents  can  move  through  ●  a  pickup  location  where  items  are  collected  ●  a  storage  or  delivery  location  where  items  must  be  placed  ●  optionally,  blocked  cells  such  as  shelves  or  occupied  spaces  
The  environment  is  primarily:

Agent  Model  and  Basic  Actions  
At  the  initial  stage,  we  assume  one  main  type  of  warehouse  agent .  Each  agent  can  perform  
one
 
action

turn
.
 
The
 
basic
 
action

is:
 
●  Move  one  square  to  an  adjacent  free  cell  ●  Pick  up  an  item  from  a  designated  pickup  location  ●  Place  an  item  at  the  required  target  location  ●  Wait  in  place  when  movement  is  not  possible  or  when  conflict  avoidance  is  needed  
Each  agent  has  a  limited  carrying  capacity.  In  the  simplest  version,  the  agent  may  carry  only  
one
 
item

a
 
time

a
 
small
 
fixed
 
number
 
of
 
items.
 
The  agent’s  main  task  is:  
1.  receive  or  detect  an  item  at  a  pickup  point  2.  identify  the  target  destination  of  that  item  3.  navigate  through  the  warehouse  4.  deliver  the  item  to  the  correct  target  location  
At  a  later  stage,  the  system  may  include  additional  task  types,  such  as  retrieving  an  item  from  
storage
 
and
 
bringing
 
it
 
to
 
an
 
exit
 
or
 
dispatch

Pathfinding  and  Conflict  Resolution  
Each  agent  must  be  able  to  move  from  its  current  position  to  a  target  cell  while  avoiding  blocked

Since

Two

The  pathfinding  problem  will  therefore  be  modeled  as  cooperative  movement  on  a  grid.  A  simple

Cooperative

At  the  beginning,  agents  may  operate  with  only  partial  local  vision,  for  example  seeing  a  radius

Because

Later,

For  future  expansion,  conflict  resolution  between  agents  may  include:

Learning  and  Route  Optimization  
In  the  first  stage,  agents  may  operate  with  fixed  rules  only.  In  later  stages,  the  system  can  
include

For

●  remember  previously  explored  areas  of  the  map  ●  prefer  shorter  or  less  congested  routes  ●  avoid  regions  where  collisions  happen  frequently  ●  update  internal  knowledge  when  new  boxes  block  movement  paths  
This  connects  to  the  project’s  route  optimization  component.  Even  a  simple  memory-based  
improvement
 
can

Success  Metrics  
The  success  of  the  system  will  be  measured  using  practical  warehouse  performance  metrics.  
Our
 
main
 
metrics
 
are:
 
●  Task  completion  rate  –  how  many  delivery  requests  are  completed  successfully  ●  Total  completion  time  –  how  many  turns  are  needed  to  finish  all  box  movements  ●  Agent  efficiency  –  average  number  of  useful  actions  versus  waiting  or  blocked  actions  ●  Collision  avoidance  –  whether  the  agents  successfully  avoid  occupying  the  same  cell  ●  Path  efficiency  –  how  close  the  actual  traveled  routes  are  to  short  or  optimal  routes  ●  System  throughput  –  number  of  boxes  delivered  in  a  fixed  number  of  turns  
In  extended  scenarios,  we  may  also  measure:  
●  average  delay  caused  by  congestion  ●  performance  as  the  number  of  agents  increases  ●  performance  under  changing  map  conditions  or  pickup/delivery  demand  
These  metrics  directly  support  the  assignment  requirement  to  define  utility  and  system  success  
in
 
measurable
 
terms.
